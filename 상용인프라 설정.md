# 상용인프라 설정

## 현재 문제점

- md-blog.org접속은 잘 되나 hyeon-28.md-blog.org (유저별 블로그 메인) 접속 시 https://md-blog.org/login?error=true 경로로 리다이랙트 되고 접속이 되지 않음.

## 시도한 내역

### 시도 1 — OAuth2 인가 요청 상태를 서버 세션 → 쿠키 저장 방식으로 변경

**원인 분석**

`hyeon-28.md-blog.org` 접속 시 아래 흐름이 발생하고 있었음:

1. 요청이 Spring Boot 백엔드에 도달함 (`*.md-blog.org` DNS가 ALB/백엔드를 가리키는 상황)
2. Spring Security가 비인증 요청으로 판단 → OAuth2 플로우를 시작
3. OAuth2 인가 요청 state를 **서버 메모리 세션**에 저장
4. GitHub가 `api.md-blog.org/login/oauth2/code/github` 로 콜백
5. **ECS 멀티 인스턴스 환경에서 콜백이 다른 인스턴스에 도달** → 세션에 state가 없음
6. `OAuth2FailureHandler` 실행 → `https://md-blog.org/login?error=true` 리다이렉트

**적용한 수정 사항**

| 파일 | 내용 |
|------|------|
| `auth/util/CookieUtils.java` (신규) | 쿠키 직렬화/역직렬화 유틸리티 |
| `auth/HttpCookieOAuth2AuthorizationRequestRepository.java` (신규) | OAuth state를 쿠키에 저장하는 `AuthorizationRequestRepository` 구현체 |
| `config/SecurityConfig.java` | `authorizationEndpoint().authorizationRequestRepository()` 에 쿠키 기반 구현체 주입 |
| `application-prod.properties` | `app.cookie.domain=.md-blog.org`, `app.cookie.secure=true` 추가 |
| `application-dev.properties` | `app.cookie.domain=`, `app.cookie.secure=false` 추가 |

**동작 원리**

- OAuth state를 `oauth2_auth_request` 쿠키(Java 직렬화 + Base64, `Domain=.md-blog.org`, `SameSite=None; Secure`, 180초 만료)에 저장
- GitHub 콜백 시 어느 인스턴스가 받더라도 브라우저가 쿠키를 전송하므로 state 매칭 성공
- `Domain=.md-blog.org` 이므로 `hyeon-28.md-blog.org`에서 저장된 쿠키가 `api.md-blog.org` 콜백에도 전달됨

**미해결 사항**

- `hyeon-28.md-blog.org` 가 백엔드로 라우팅되는 근본 원인(DNS/CloudFront 설정)은 인프라 변경 필요
  - `*.md-blog.org` 를 React SPA(프론트엔드)로 라우팅하도록 CloudFront 배포에 Alternate Domain Name 추가 권장
  - 현재 수정으로 OAuth 실패는 해소되나, 서브도메인 접속 시 React SPA가 아닌 Spring Boot 응답이 반환될 수 있음

### 시도 2 — 서브도메인 요청을 프론트엔드 경로로 리다이렉트 (코드 레벨 완전 해결)

**목표**: DNS/CloudFront 변경 없이 코드만으로 `hyeon-28.md-blog.org` 접속 시 블로그가 보이도록 수정

**적용한 수정 사항**

| 파일 | 내용 |
|------|------|
| `auth/filter/SubdomainRedirectFilter.java` (신규) | Spring Security보다 먼저 실행 (`HIGHEST_PRECEDENCE`), 유저 서브도메인 감지 시 `{frontendUrl}/blog/{username}` 으로 302 리다이렉트 |
| `application-prod.properties` | `app.root.domain=md-blog.org` 추가 |
| `src/App.tsx` | `/blog/:username` 라우트 추가 (`BlogMainRoute` 컴포넌트), 해당 경로도 auth 로딩 없이 즉시 `BlogMain` 렌더링 |

**최종 접속 흐름**

```
브라우저 → hyeon-28.md-blog.org
  └→ ALB → Spring Boot 백엔드
      └→ SubdomainRedirectFilter (Spring Security보다 먼저 실행)
          └→ 302 redirect → https://md-blog.org/blog/hyeon-28
              └→ CloudFront → S3 → React SPA
                  └→ /blog/:username 라우트 → BlogMain(username="hyeon-28")
```

**보존된 동작**

- 서브도메인이 직접 CloudFront(프론트엔드)로 라우팅되는 경우에도 기존 `getBlogUsername()` 로직이 그대로 동작함 (두 경로 모두 지원)
- `api.md-blog.org`, `www.md-blog.org` 는 리다이렉트 제외 대상으로 처리
