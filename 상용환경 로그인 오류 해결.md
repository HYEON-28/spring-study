# 상용환경 로그인 오류 해결

## 현재 문제점

md-blog.org로 접속은 되나 로그인 시도 시

```
지금 이 페이지가 작동하지 않습니다.
api.md-blog.org 너무 여러 번 리디렉션되었습니다.

이 사이트의 쿠키를 삭제해 보세요.
```

이런 오류가 뜸
원랜 문제가 없었으나 "상용인프라 설정.md"작업 이후 문제 생김

## 시도한 내역

### 시도 1 — `/error` permitAll 누락으로 인한 리다이렉트 루프 차단

**원인 분석**

`상용인프라 설정.md` 작업으로 추가된 `HttpCookieOAuth2AuthorizationRequestRepository`가
`OAuth2AuthorizationRequest` 직렬화/역직렬화 중 `RuntimeException`(예: `IllegalArgumentException`)을
던지면 다음 루프가 발생함:

1. `/oauth2/authorization/github` 또는 `/login/oauth2/code/github` 처리 중 `RuntimeException` 발생
2. `AbstractAuthenticationProcessingFilter`는 `AuthenticationException`만 catch — `RuntimeException`은 그대로 전파
3. `ExceptionTranslationFilter`도 `RuntimeException`은 처리 안 함 → 컨테이너로 전파 → `/error`로 forward
4. `SecurityConfig`의 `permitAll` 목록에 `/error`가 없음 → `anyRequest().authenticated()` 매칭 → `AccessDeniedException`
5. `ExceptionTranslationFilter`가 `LoginUrlAuthenticationEntryPoint` 호출 → `/oauth2/authorization/github`로 302
6. → 1로 돌아감 → **api.md-blog.org에서 무한 리다이렉트 루프**

**적용한 수정 사항**

| 파일 | 내용 |
|------|------|
| `config/SecurityConfig.java` | `permitAll` 목록에 `/error` 추가 (방어적 안전장치) |
| `auth/HttpCookieOAuth2AuthorizationRequestRepository.java` | `loadAuthorizationRequest`/`saveAuthorizationRequest`에서 `RuntimeException` catch 후 로그만 남기고 진행 — 직렬화 실패 시 `OAuth2FailureHandler`(→ `/login?error=true`)로 정상 분기 |

**미해결 (후속 조치 필요)**

- 위는 "루프 차단" 안전장치이며, 직렬화 실패의 **근본 원인**은 별도로 조사 필요. 상용 배포 후 ECS 로그에서
  `oauth2_auth_request 쿠키 역직렬화 실패` / `OAuth2 authentication failed` 메시지로 실제 원인 확인 가능
- 직렬화가 실패하지 않는 정상 흐름이라면 이 수정 없이도 로그인은 동작함
