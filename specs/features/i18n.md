# Feature: 다국어(i18n) 지원

**Status:** 구현 완료 (일부 페이지)

## 목적

사용자가 선호하는 언어를 선택하면 UI 텍스트가 해당 언어로 표시되고, 페이지 이동 후에도 유지된다.

## 지원 언어

| 코드 | 언어            |
| ---- | --------------- |
| `ko` | 한국어 (기본값) |
| `en` | English         |
| `ja` | 日本語          |
| `zh` | 中文            |

## 동작 방식

1. `localStorage` 키 `md-blog.lang` 에서 언어 설정 읽기
2. 유효하지 않은 값이면 기본값 `ko` 사용
3. 언어 변경 시 localStorage 업데이트 + `document.documentElement.lang` 변경
4. `StorageEvent` 리스닝으로 같은 브라우저 다른 탭과 동기화

## 수용 기준

- [x] 언어 선택 후 다른 페이지로 이동해도 유지 (localStorage 영속)
- [x] 브라우저 새로고침 후에도 유지
- [x] 지원하지 않는 언어 코드는 무시하고 기본값 사용

## i18n 적용 현황

| 페이지/컴포넌트 | 파일               | 적용 여부            |
| --------------- | ------------------ | -------------------- |
| Landing         | `i18n/landing.ts`  | ✅                   |
| Login           | `i18n/login.ts`    | ✅                   |
| Nav             | `i18n/nav.ts`      | ✅                   |
| Footer          | `i18n/footer.ts`   | ✅                   |
| RepoLink        | `i18n/repolink.ts` | ✅                   |
| Main (대시보드) | —                  | ❌ (한국어 하드코딩) |
| RepoSettings    | —                  | ❌                   |
| BlogSettings    | —                  | ❌                   |
| FileUpdated     | —                  | ❌                   |

## 구현 구조

```
src/i18n/
├── index.ts        # Lang 타입 정의: "ko" | "en" | "ja" | "zh"
├── landing.ts      # Landing 페이지 텍스트
├── login.ts        # Login 페이지 텍스트
├── nav.ts          # Nav 컴포넌트 텍스트
├── footer.ts       # Footer 컴포넌트 텍스트
└── repolink.ts     # RepoLink 페이지 텍스트
```

각 파일 구조:

```typescript
export const LANDING_I18N: Record<Lang, { ... }> = {
  ko: { ... },
  en: { ... },
  ja: { ... },
  zh: { ... },
}
```

- `LangContext` — 전역 언어 상태 (`lang`, `setLang`)
- `useLang()` 훅으로 어디서든 현재 언어 접근

## 미적용 페이지 처리 방침

Main, RepoSettings, BlogSettings, FileUpdated는 현재 한국어로 하드코딩되어 있음.
새 텍스트 추가 시 i18n 파일에 먼저 작성하고 `useLang()`으로 연결한다.
